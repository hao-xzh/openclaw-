# 脚本使用教程

这份文档只讲一件事：怎么使用仓库里的“一键脚本”，以及脚本报错时怎么处理。

如果你只是想安装、接入 DeepSeek，或者卸载 OpenClaw，看这份文档就够了。

## 先看你要用哪个脚本

Windows 使用这 3 个文件：

- `Win一键安装.bat`
- `Win一键接入DeepSeek.bat`
- `Win一键卸载.bat`

mac 使用这 3 个文件：

- `Mac一键安装.command`
- `Mac一键接入DeepSeek.command`
- `Mac一键卸载.command`

## 用脚本安装 OpenClaw

### 在 Windows 上安装

双击：

```powershell
Win一键安装.bat
```

如果你想在终端里运行：

```powershell
.\Win一键安装.bat
```

常用参数：

```powershell
.\Win一键安装.bat -Proxy "http://127.0.0.1:7890"
.\Win一键安装.bat -NoOnboard
.\Win一键安装.bat -NoNode
.\Win一键安装.bat -InstallMethod npm
.\Win一键安装.bat -Verbose
.\Win一键安装.bat -DryRun
```

### 在 mac 上安装

双击：

```bash
Mac一键安装.command
```

如果你想在终端里运行：

```bash
bash Mac一键安装.command
```

常用参数：

```bash
bash Mac一键安装.command --proxy http://127.0.0.1:7890
bash Mac一键安装.command --no-onboard
bash Mac一键安装.command --no-node
bash Mac一键安装.command --install-method npm
bash Mac一键安装.command --verbose
bash Mac一键安装.command --dry-run
```

## 用脚本接入 DeepSeek

先完成 OpenClaw 安装，再运行 DeepSeek 接入脚本。

### 在 Windows 上接入

双击：

```powershell
Win一键接入DeepSeek.bat
```

或者在终端里运行：

```powershell
.\Win一键接入DeepSeek.bat
```

脚本会依次做这些事：

1. 检查 OpenClaw 是否已经安装。
2. 让你输入 DeepSeek API Key。
3. 让你选择默认模型。
4. 把配置写入 `~/.openclaw/openclaw.json`。
5. 尝试验证 API 是否可连通。

### 在 mac 上接入

双击：

```bash
Mac一键接入DeepSeek.command
```

或者在终端里运行：

```bash
bash Mac一键接入DeepSeek.command
```

## 用脚本卸载 OpenClaw

### 在 Windows 上卸载

```powershell
Win一键卸载.bat
```

或者在终端里运行：

```powershell
.\Win一键卸载.bat
```

## 遇到报错先这样排查

先确认两件事：

1. 你运行的是当前仓库里的最新脚本，不是之前复制出去的旧版本。
2. 脚本文件名没有被系统自动改名，比如加了 `(1)`、`副本`、下载时间后缀。

如果是双击闪退，先不要继续双击，改成在终端里运行，这样能看到完整报错。

Windows：

```powershell
cmd /k Win一键安装.bat
```

或者：

```powershell
cmd /k Win一键接入DeepSeek.bat
```

mac：

```bash
bash Mac一键安装.command
```

或者：

```bash
bash Mac一键接入DeepSeek.command
```

## 常见错误和解决办法

### 提示找不到 PowerShell

常见报错：

```text
[ERROR] PowerShell was not found on this system.
```

这说明当前 Windows 环境里没有可用的 `powershell` 命令。

先试：

```powershell
powershell
```

如果命令本身都不存在，说明系统环境异常，先修复 PowerShell，再运行脚本。

### 双击后乱码，或者窗口一闪而过

先不要继续双击，改成：

```powershell
cmd /k Win一键安装.bat
```

如果你看到的是旧的乱码报错，通常是旧版本脚本还在被执行。把旧文件删掉，只保留最新的中文命名脚本，再试一次。

### 提示“不是内部或外部命令”

这类报错通常不是脚本逻辑问题，而是：

- 运行了旧版本脚本
- 文件损坏
- 文件名被改了，但你执行的还是旧名字

先确认当前目录里实际存在这些文件：

```powershell
dir
```

然后直接运行：

```powershell
.\Win一键安装.bat
```

不要再运行旧名字。

### mac 提示没有权限运行 `.command`

先执行：

```bash
chmod +x Mac一键安装.command
chmod +x Mac一键接入DeepSeek.command
chmod +x Mac一键卸载.command
```

然后再运行：

```bash
bash Mac一键安装.command
```

如果是 Finder 双击时报系统安全提示，按住 `Control` 再点脚本，选择“打开”。

### 提示 OpenClaw 未安装

这通常发生在运行 DeepSeek 接入脚本时。

先安装：

```powershell
.\Win一键安装.bat
```

或者：

```bash
bash Mac一键安装.command
```

安装完成后再运行 DeepSeek 接入脚本。

### 提示 Node.js 未安装

安装脚本默认会自动处理 Node.js。

如果你自己加了跳过参数：

Windows：

```powershell
.\Win一键安装.bat -NoNode
```

mac：

```bash
bash Mac一键安装.command --no-node
```

那就需要你先手动安装 Node.js，再重新运行脚本，或者去掉这个参数。

### 提示网络连接失败、下载超时、GitHub 不可达

先使用VPN然后加代理再运行。

Windows：

```powershell
.\Win一键安装.bat -Proxy "http://127.0.0.1:7890"
```

mac：

```bash
bash Mac一键安装.command --proxy http://127.0.0.1:7890
```

如果没有代理，也可以先运行安装脚本默认流程。脚本会自动尝试官方源和镜像源。

### DeepSeek 接入时提示 API Key 不正确

先检查：

1. Key 是否完整。
2. 是否以 `sk-` 开头。
3. 账号余额和权限是否正常。

然后重新运行：

```powershell
.\Win一键接入DeepSeek.bat
```

或者：

```bash
bash Mac一键接入DeepSeek.command
```

## 推荐的最简使用方式

如果你不想记太多，只记下面这几个文件就行。

Windows：

- 安装：`Win一键安装.bat`
- 接入 DeepSeek：`Win一键接入DeepSeek.bat`
- 卸载：`Win一键卸载.bat`

mac：

- 安装：`Mac一键安装.command`
- 接入 DeepSeek：`Mac一键接入DeepSeek.command`
- 卸载：`Mac一键卸载.command`

出问题时，不要先双击，先在终端里运行。这样最容易看到真实报错。
